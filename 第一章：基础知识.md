### 基础知识

>   汇编语言是直接在硬件之上工作的编程语言，我们首先要了解硬件系统的结构，才能有效地应用汇编语言对其编程。在汇编课程中我们不对硬件系统进行全面和深入的研究，这不在这门课的范围内。
>
>   关于 PC 机及 CPU 物理结构和编程结构的研究，在《微机原理与接口》中进行；
>
>   对于计算机一般的结构、功能、性能的研究在《计算机组成原理》中进行；
>
>   汇编课程的研究重点放在如何利用硬件系统的编程结构和指令集有效灵活地控制系统进行工作。
>
>   *《汇编语言》write by 王爽*

#### 汇编指令

汇编语言的主体是汇编指令。汇编指令和机器指令的差别在于指令的表示方法上。汇编指令是机器指令便于记忆的书写格式，例如：

-   操作：寄存器 BX 的内容送到 AX 中
-   机器指令：1000100111011000
-   汇编指令：mov  ax,  bx

汇编语言的核心是汇编指令，它决定了汇编语言的特性，发展至今主要又三类指令组成：

-   汇编指令：机器码的助记符，有对应的机器码。
-   伪指令：没有对应的机器码，由编译器执行，计算机并不执行。
-   其他符号：如+、-、*、/ 等，由编译器识别，没有对应的机器码。

#### 指令与数据

1.  CPU 是计算机的核心部件，想让CPU 工作就必须给它提供指令和数据，而指令和数据在存储器中存放，也就是通常意义上的内存。
2.  在内存中，指令和数据没有任何区别，都是二进制信息（010101···），CPU 在工作的时候有的时候将它看作指令，有的时候将它看作数据。

#### CPU对存储器的读写

​		计算器的最小单位是`bit`(比特)，8个`bit`组成一个`byte`，微机存储器的容量是以`byte`为最小单位来计算的，因此对于拥有128个存储单元的存储器，我们可以说它的容量是128字节。同理，平常所说的内存容量也是以`byte`为最小单元，其中的单位换算如下：

-   `1KB` = `1024B`、`1MB` = `1024KB`、`1GB` = `1024MB`、`1TB` = `1024GB`

---

CPU 要想进行数据的读写，必须和外部器件进行三类信息的交互：

+   存储单元的地址（地址信息）
+   器件的选择，读或者写的命令（控制信息）
+   要读或者写的数据（数据信息）

#### 内存地址空间

![各类存储器逻辑连接](C:%5CUsers%5CKarene%5CDesktop%5Ctemp%5Cdoc%5C8086asm%5Csrc%5C%E7%AC%AC%E4%B8%80%E7%AB%A0%EF%BC%9A%E5%9F%BA%E7%A1%80%E7%9F%A5%E8%AF%86%5Cmemory-logic.png)

PC 机本身的内存包含了很多部分，从硬件上来看，组成整个内存空间的硬件包括了两类：

-   随机存储器`RAM`（Random Access Memory）：断电即消失
-   只读存储器`ROM`（Read Only Memory）：断电不消失

这里的存储器指的是内存的类型，当然现在的内存技术发展迅速，但是我们在这里并不讨论具体的技术，只是将知识抽象来理解，贯彻我们的宗旨：用到什么学什么。

![看作一个逻辑存储器](C:%5CUsers%5CKarene%5CDesktop%5Ctemp%5Cdoc%5C8086asm%5Csrc%5C%E7%AC%AC%E4%B8%80%E7%AB%A0%EF%BC%9A%E5%9F%BA%E7%A1%80%E7%9F%A5%E8%AF%86%5Cmemory-union.png)

上述的存储器，在物理上都是独立的器件，但是有两点是相同的：

-   都和 CPU 的总线相连。
-   CPU 对他们进行读写时通过控制线发出读写命令。

​		也就是说 CPU 在面对他们时将这些独立的器件当作统一的内存来对待，这个统一看待的对象在逻辑上看就像是一个整体的**内存地址空间**。这样一来，我们就可以知道向一个地址读写数据时，实际操作的对象究竟是谁。

因此我们在基于一个计算机硬件系统进行编程的时候，必须知道这个系统中的内存地址分配情况，因为当我们想在某类存储器中读写数据的时候，必须知道它的第一个单元的地址和最后一个单元的地址，才能保证读写操作是在预期的存储器中进行。比如，我们希望向显示器输出一段信息，那么必须将这段信息写入到显存中，现存才能将它输出到显示器上。要向显存写入数据就必须知道显存在内存地址空间的地址。

![8086内存地址空间分配](C:%5CUsers%5CKarene%5CDesktop%5Ctemp%5Cdoc%5C8086asm%5Csrc%5C%E7%AC%AC%E4%B8%80%E7%AB%A0%EF%BC%9A%E5%9F%BA%E7%A1%80%E7%9F%A5%E8%AF%86%5C8086-memory-space.png)

>   最终运行程序的是CPU，我们用汇编语言编程的时候，必须要从CPU的角度考虑问题。对CPU来讲，系统中所有的存储器中的存储单元都处于一个统一的逻辑存储器中，它的容量受CPU寻址能力的限制。这个逻辑存储器就是我们所说的内存地址空间
>
